# 二、运行和打包部署

# 2.1 环境配置

https://nodejs.org/en/download/ node js 官网下载，会自动配置 npm node版本需要在14.15.0以上，17以下。因为项目中node-sass版本问题，需要与node版本相匹配，否则会报兼容性错误。 如果本地开发环境更新了 node 版本，对应的 node_modules 也要重新下载，不然会报兼容性问题

如node-sass不兼容本地node，可以执行如下命令。

npm rebuild node-sass

# 2.2 安装依赖

npm install | yarn | pnpm install 注意：下载 npm 包的时候是根据系统里边的.npmrc 文件指定的下载地址来下载的，如果不想修改系统的配置，可以在项目根目录新建.npmrc文件配置镜像源 网速问题导致包没有下载完全导致报错，重新执行 npm install 下载还不行的话，再删除 node_modules 目录重新下载试试

# 2.3 运行项目

npm run dev

# 2.4 打包项目

目前项目中的打包均采用了自建工具打包，详情可以查看项目根目录下的deploy-server.js。 node deploy-server.js | npm run deploy 可以打开打包配置页面，选择需要打包的环境。
